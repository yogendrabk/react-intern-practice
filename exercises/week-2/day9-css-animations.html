Pattern 1: Button Hover Transitions

💡 What Learn: CSS transitions, smooth property change
TRANSITION vs ANIMATION - Key Difference:
  • TRANSITION: Smooth change from state A to B on event (hover, click)
    Example: transition: background 0.3s on hover change color
  • ANIMATION: Runs automatically using @keyframes, doesn't need event
    Example: animation: spin 1s infinite keep spinning forever

This pattern use TRANSITION → Hover button, smooth color+scale+shadow change.

Code Explanation: 
transition: background 0.3s ease, transform 0.3s ease, box-shadow 0.3s ease;
/* What to animate | Duration | Timing function */

:hover {
    background: #764ba2;      /* Change color */
    transform: scale(1.05);   /* Grow 5% */
    box-shadow: 0 8px 16px;   /* Add shadow */
}

transition make change smooth instead of instant. 0.3s mean animation take 0.3 second. ease mean start slow-end slow.

Pattern 2: Loading Spinner Animation

💡 What Learn: @keyframes, animation loop
Using @keyframes for continuous animation: 
@keyframes spin {
    0% { transform: rotate(0deg); }      /* Start: 0 degree */
    100% { transform: rotate(360deg); }   /* End: 360 degree (full circle) */
}

.spinner {
    animation: spin 1s linear infinite;
    /* What animation | Duration | Speed | Loop forever */
}

@keyframes spin define animation. 0% = start, 100% = end. infinite keep looping forever. Perfect for loading screen!

Pattern 3: Pulsing Notification Dot

💡 What Learn: Complex @keyframes with multiple steps
Keyframe with 3 step (0%, 50%, 100%): 
@keyframes pulse {
    0%   { transform: scale(1);   opacity: 1; }   /* Normal size, visible */
    50%  { transform: scale(1.2); opacity: 0.7; } /* Grow, fade slightly */
    100% { transform: scale(1);   opacity: 1; }   /* Back to normal */
}

Pattern 4: Card Flip Animation (3D)

💡 What Learn: 3D transform, perspective, preserve-3d
3D Animation with perspective:
  • perspective: 1000px — Add depth, make 3D effect realistic
  • transform-style: preserve-3d — Child element in 3D space (not flat)
  • transform: rotateY(180deg) — Rotate on Y axis (vertical flip)
  • backface-visibility: hidden — Hide back side of element when flipped

    transition: transform 0.6s;

.flip-card:hover .flip-card-inner {

}

.flip-card-back {
    backface-visibility: hidden;  /* Hide back when facing away */
    transform: rotateY(180deg);   /* Pre-rotate back 180deg */
}

Timing Functions Comparison

💡 Different animation speed profile
Timing functions control animation speed:
  • ease — Slow start, fast middle, slow end (smooth, natural)
  • linear — Same speed throughout (mechanical)
  • ease-in — Slow start, then fast (accelerating)
  • ease-out — Fast start, then slow (decelerating)

🛡️ Defense Questions & Answers

Q: CSS transition ra animation ko difference ke ho? (What's difference between transition and animation?)

A: Transition: Smooth change from state A to B when event happen (like hover). Need trigger. Example: Button color change on hover.
Animation: Run independently using @keyframes. No event needed, can loop forever. Example: Spinning loader that auto-rotate.

Q: @keyframes kina use garchau? (Why use @keyframes?)

A: @keyframes define animation step. 0% mean start state, 100% mean end state. Can have middle step (50%, 75%, etc). Browser auto-interpolate between keyframe, creating smooth animation.

Q: perspective ra transform-style: preserve-3d ko purpose ke ho? (What do they do?)

A: perspective: 1000px add 3D depth effect (farther 1000px). transform-style: preserve-3d tell browser child element in 3D space, not flat. Together make card flip look realistic with depth!

Q: Skeleton loader kina useful? (Why skeleton loader useful?)

A: Skeleton loader show placeholder layout while real data loading. Much better than blank page or spinner. User see structure, understand what loading. Result: Perceived faster loading, better UX. Very common in React apps!

Q: Timing function kina matter? ease vs linear ke difference? (Why timing function important?)

A: Timing function control animation speed curve. ease natural, smooth, good for most case. linear mechanical, robotic. ease-in/out special effect. Choosing right timing make animation feel professional and natural!
